# North Star Website — Technical Strategy

This document is the technical reference for building and maintaining the North Star marketing website. All architectural decisions, framework choices, and tooling should align with what's documented here.

---

## Stack

| Layer | Technology |
|-------|-----------|
| Framework | Astro v5 (static site generation) |
| Styling | Tailwind CSS v4 (via `@tailwindcss/vite` plugin) |
| Language | TypeScript (strict mode) |
| Deployment | Vercel (auto-deploy from Git) |
| Sitemap | `@astrojs/sitemap` |
| Formatting | Prettier with `prettier-plugin-astro` + `prettier-plugin-tailwindcss` |

---

## Architecture Principles

**Static-first.** Every page is pre-rendered to HTML at build time. No server-side rendering, no API routes, no database queries. The output is a folder of HTML, CSS, and images that can be served from any CDN.

**Zero JavaScript by default.** Astro strips all JS from components unless explicitly opted in with `client:*` directives. The only JS on the page is for dark mode toggle and pricing toggle — delivered as Astro islands, not full-page hydration.

**Component-based.** Each page section is an Astro component in `src/components/`. The page assembles them in `src/pages/index.astro`. This keeps each section self-contained and easy to modify independently.

## Design System

### Colors

CSS custom properties defined in `src/styles/global.css`. All components reference these variables so dark mode is a single toggle.

**Light mode:**
- Background: `#ffffff`
- Surface: `#f8fafc` (slate-50)
- Text primary: `#0f172a` (slate-900)
- Text secondary: `#475569` (slate-600)
- Accent: `#3b82f6` (blue-500)
- Accent hover: `#2563eb` (blue-600)
- Border: `#e2e8f0` (slate-200)

**Dark mode:**
- Background: `#0f172a` (slate-900)
- Surface: `#1e293b` (slate-800)
- Text primary: `#f8fafc` (slate-50)
- Text secondary: `#94a3b8` (slate-400)
- Accent: `#60a5fa` (blue-400)
- Accent hover: `#3b82f6` (blue-500)
- Border: `#334155` (slate-700)

### Typography

System font stack for zero font-loading cost:

```css
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif;
```

**Scale:**
- Hero headline: `text-5xl` / `text-6xl` (48px / 60px)
- Section headline: `text-3xl` / `text-4xl` (30px / 36px)
- Body: `text-base` / `text-lg` (16px / 18px)
- Small: `text-sm` (14px)

### Spacing

Use Tailwind's default spacing scale. Section padding: `py-20` (80px) on desktop, `py-12` (48px) on mobile. Max content width: `max-w-6xl` (1152px) centered with `mx-auto px-6`.

### Border Radius

Cards and buttons: `rounded-xl` (12px). Smaller elements: `rounded-lg` (8px).

## Dark Mode Strategy

1. On page load, check `localStorage.getItem("theme")`.
2. If no stored preference, respect `prefers-color-scheme` media query.
3. Apply `class="dark"` to `<html>` element for dark mode.
4. All color references use Tailwind's `dark:` variant.
5. The `ThemeToggle` component (an Astro island with inline `<script>`) toggles the class and persists to `localStorage`.
6. To prevent flash of wrong theme, the theme detection script runs in `<head>` as a blocking inline script.

---

## Image Strategy

- Product screenshots and hero images go in `src/assets/images/` for Astro's build-time optimization.
- SVG icons go in `src/assets/icons/` and are inlined in components.
- Favicons and OG images go in `public/` (served as-is).
- Use Astro's `<Image>` component for automatic WebP conversion and responsive `srcset`.
- Lazy-load images below the fold with `loading="lazy"`.

## SEO

- Page `<title>` and `<meta name="description">` set in `BaseLayout.astro`.
- Open Graph tags (`og:title`, `og:description`, `og:image`, `og:url`) for social sharing.
- Twitter card meta tags.
- Canonical URL via Astro's `site` config.
- Sitemap generated by `@astrojs/sitemap` at `/sitemap-index.xml`.
- `robots.txt` in `public/`.

## Deployment

Vercel auto-deploys from the `main` branch. Preview deploys are created for pull requests.

Build command: `npm run build`
Output directory: `dist/`

No environment variables are required for the marketing site. If analytics are added later, the analytics key would be the only env var.
